Introduction

Documentation for the NISSINObjects library written in google apps scripts by yours truly.

Functions:

buildObjects(): updates / builds all the shipments (including lclShipments, fclShipments, and coloadShipments) and returns allShipments

buildLCLs(): updates and returns lclShipments

buildColoads(): updates and returns coloadShipments

buildFCLs():updates and returns fclShipments

getLCLShipmentFromHBL(hblNumber, lclShipments): returns the matching shipment object from the lclShipments objects provided

getColoadShipmentFromHBL(hblNumber, coloadShipments): returns the matching shipment object from the coloadShipments objects provided

getFCLShipmentFromHBL(hblNumber, fclShipments): returns the matching shipment object from the fclShipments objects provided

getShipmentFromHBL(hblNumber, allShipments, shipmentTypes = ['lcl', 'fcl', 'coload']): returns the matching shipment object from the allShipments objects provided. shipmentTypes is an optional argument to limit which types are searched through to match HBL numbers. Mostly useless.

getAllLCLHBLs(lclShipments): returns an array of all hbl objects in the given lclShipments object provided

getAllColoadHBLs(coloadShipments): returns an array of all hbl objects in the given coloadShipments object provided

getAllFCLHBLs(fclShipments): returns an array of all hbl objects in the given fclShipments object provided

getAllHBLs(allShipments, shipmentTypes = ['lcl', 'fcl', 'coload']): returns an array of all hbl objects in the given allShipments object provided. shipmentTypes is an optional argument to limit which types are searched through and appended to the returned hbl array. Potentially useful.

Usage:

An example of how this is used:


    NISSINObjects.buildObjects();

    var shipments = NISSINObjects.allShipments;

The first line updates / builds the objects, and the second utilizes the global variable within NISSINObjects. You can achieve the same functionality with the following:

  var shipments = NISSINObjects.buildObjects();

But the buildObjects function is purposely decoupled so you do not have to re-build all shipments every time. And saves you from keeping track / creating your own global variables for shipment types.

Ex:


    // Update / build objects ONCE
    NISSINObjects.buildObjects();
                
    var lcls = NISSINObjects.lclShipments;
    // Process lcls for something

    var fcls = NISSINObjects.fclShipments;
    // Process fcls for something

    var coloads = NISSINObjects.coloadShipments;
    // Process coloads for something

Using the optional parameters:


    // get all HBLs but limit to lcls and coloads (exclude fcls)
    NISSINObjects.getAllHBLs(allShipments, ["lcl", "coload"]);

    // get all HBLs, don't limit types
    NISSINObjects.getAllHBLs(allShipments);
